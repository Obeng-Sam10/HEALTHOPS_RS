Rust CLI Automation Log + Network Monitoring

HealthOps_rs

A lightweight command-line tool that helps operators detect issues early by scanning log files and monitoring network connections — built with Unix-style text processing in Rust.

Download How it works Pricing GitHub Repo

Problem

Small clinics, labs, and student projects often rely on manual checking of system logs and network activity. This is slow and people miss warnings. HealthOps_rs automates the core checks and prints clear results and alerts.

What it does

Log Scanning

Find and count errors in log files (grep/wc ideas).

  • Counts total lines
  • Counts occurrences of ERROR
  • Warns when errors exceed a threshold

Network Monitoring

Checks active connections by reading OS output (system programming).

  • Reads connection list from the OS
  • Counts connections (approximation)
  • Warns on unusually high activity

Developer Workflow

One command to run repeatable checks and produce consistent output.

  • Fast CLI usage
  • Simple commands
  • Easy to extend

How it works (simple)

  1. You type a command in the terminal.
  2. HealthOps_rs reads files / asks the OS for network info.
  3. It processes the output as text (lines, matches, counts).
  4. It prints a short report and warnings using simple rules.

Example commands

# Help
cargo run help

# Scan logs (uses test.log in project root)
cargo run scan logs

# Scan network connections
cargo run scan network

Note: released binaries are Linux executables. On macOS you can download/extract them but you cannot run them directly.
